Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo, ao utilizar esse modelo, ele pode ser explicado pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 7 - Busca em Matriz¶

O que a tcnica de busca em matriz (grid search) e como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Resposta: Busca em matriz uma tcnica utilizada para o modelo de regresso logistica. Ela se baseia em utilizar-se de hiperparametros para se chegar a valores de preciso, utilizando o modelo e os dados de validao( alguma tcnica de validao cruzada). Com o resultado, obtem-se os parmetros, que seriam os valores utilizados no modelo, a partir de um hiperparametro(tambm chamado como meta parmetro).

o busca em matriz funciona assim: So treinados varios modelos, cada um com algoritmos e hiperparametros distintos. Cada modelo receber uma nota (ou score), dependendo da performance em relao ao algoritmo e hiperparmetro. No final do processo, ns selecionamos o algoritmo com o hiperparmetro que teve o score maior.

Aps a escolha, o modelo utilizado com os dados de teste para garantir que est de acordo.

Questão 8 - Validação Cruzada¶

O que a tcnica de treinamento de validao-cruzada k-fold? Quais benefcios essa tcnica proporciona para busca em matriz ao otimizar um modelo?
Dica: Assim como h um raciocnio por trs de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada?

Resposta: A tecnica k-fold visa o reuso do grupo de dados utilizados para treino e teste. Apenas utilizando a tcnica de validao cruzada, no possvel reutilizar o grupo de dados novamente, para no 'viciar' o modelo, uma vez que ele j conhece os dados utilizados, sendo necesrio descarta-los.

No k-fold, os dados so divididos K vezes. Cada diviso(ou bucket) possui uma quantidade de dados que igual em todos os buckets. Antes de escolher os dados de um bucket, eles so sortidos(shuffle). O modelo tambm testado K vezes, e em cada teste utilizado um bucket diferente. E depois de testado, feito a mdia dos resultados para se obter o modelo final.

O problema que surgiria ao se utilizar a busca em matriz sem utilizar um conjunto de validao cruzada que a parametrizao(calibrao) da busca em matriz ia utilizar os dados j utilizados por parmetros anteriores, para validar o nvel de preciso de um hiperparametro. Com isso, ao final do processo de busca em matriz, o modelo no estaria adequado, pois ele se especializou(ou 'viciou') nos dados utilizados na busca, e deixaria de ter um desempenho satisfatrio em dados desconhecidos. Em resumo, no se pode testar um modelo no qual dos dados de teste j foram utilizados para o treinamento (e para a busca em grade).

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth': range(1, 11)}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)
    
    # TODO: Gerar o objeto de busca em matriz    
    grid = GridSearchCV(regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_

Uma vez que o modelo foi treinado em conjunto de dados atribudo, ele agora pode ser utilizado para fazer estimativas em novos conjuntos de entrada de dados. No caso do regressor da rvore de deciso, o modelo aprendeu quais so as melhores perguntas sobre a entrada de dados, e pode responder com uma estimativa para a varivel alvo. Voc pode utilizar essas estimativas para conseguir informaes sobre os dados dos quais o valor da varivel alvo desconhecida por exemplo, os dados dos quais o modelo no foi treinado.
